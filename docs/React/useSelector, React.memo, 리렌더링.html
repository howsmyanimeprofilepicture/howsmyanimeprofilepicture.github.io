3. useSelector의 리렌더링

3.1. useSelector로 object가 아닌 값을 return 하는 경우

​x
import React from 'react'
import { useSelector } from 'react-redux'
​
export const CounterComponent = () => {
  const counter = useSelector(state => state.counter)
  return <div>{counter}</div>
}
  • 뇌피셜인데... useSelector는 리덕스가 관리하는 state에 어떠한 변화가 생기면, 그러니깐 새로운 dispatch가 발생하면 ===연산자를 통해 리턴값을 비교하는 것 같다.
  • 위 예시에서 useSelector는 state.counter를 반환한다. 즉 (객체가 아닌 원시값)숫자를 반환한다.
  • 그래서 counter가 아닌 다른 state가 변경됐다면 이전counter===지금counter결과는 여전히 true이므로 리렌더링은 발생하지 않는다.

 

 

3.2. useSelector로 object를 return 하는 경우

xxxxxxxxxx
import React from 'react'
import { useSelector } from 'react-redux'
​
export const CounterComponent = () => {
  const {counter} = useSelector(state => ({counter : state.counter}))
  return <div>{counter}</div>
}

  • 하지만 useSelector가 객체를 반환한다면 이야기가 달라진다.

  • counter가 아닌 다른 state를 변화시키는 dispatch가 발생해도 위 예시의 useSelector는 새로운 객체를 반환한다.

  • 새롭게 반환된 객체의 counter값은 동일하지만, 새롭게 반환된 객체에 할당된 메모리 주소가 달라서 기존 객체===새롭게 반환된 객체 의 결과는 false가 된다.

  • ===결과가 false가 되면 스테이트에 변화가 있다고 판단해서 리렌더링이 된다.

  • 위 예시해서 counter가 아닌 다른 state가 변한 경우에도 해당 컴포넌트(CounterComponent)는 리렌더링 된다.

  • xxxxxxxxxx
    const {counter} = useSelector(state => {counter : state.counter}, shallowEqual)
    // 이렇게 shallowEqual 함수를 두번째 인자로 전달해서 해결 가능

  • useSelector의 2번째 인자로 shallowEqual 함수를 전달하면, 반환하는 객체 자체의 내용 자체가 같다면 리렌더링을 일으키지 않는다.
