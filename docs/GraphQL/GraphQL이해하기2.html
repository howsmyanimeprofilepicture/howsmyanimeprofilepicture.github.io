1) 쿼리와 뮤테이션의 차이는 "없다."

  • 조금 놀라울 수 있는 사실인데, 쿼리와 뮤테이션의 차이는 없다. 약간 컨벤션적이고 신텍스적으로 구분할 뿐이다.

  • 쿼리도 Resolver 함수를 통해 (서버안에서 다뤄지는) 값을 수정할 수 있고, 뮤테이션도 Return값을 갖기 때문이다.

  • 물론 리액트 훅 useMutation와 useQuery의 차이는 존재한다. useMutation은 해당 Mutation을 요청할 수 있는 함수를 리턴하고,

  • useQuery는 해당 컴포넌트의 렌더링에 따라 자동으로 요청된다는 정도의 차이이다.

     

When your component renders, useQuery returns an object from Apollo Client that contains loading, error, and data properties you can use to render your UI.

출처 : https://www.apollographql.com/docs/react/data/queries/

 

2) Operation Name Query(작업 이름 쿼리)

  • 쿼리는 일반 쿼리(단축 문법)와 오퍼레이션 네임 쿼리(a.k.a. 작업 이름 쿼리)가 있다.

  • # 일반 쿼리 (단축 문법)
    {
      hero {
        name
        friends {
          name
        }
      }
    }
    # 오퍼레이션 네임 쿼리
    query HeroNameAndFriends {
      hero {
        name
        friends {
          name
        }
      }
    }

  • 위 예제에서 query는 작업 타입(Operation Type)이다.

  • HeroNameAndFriends는 작업 이름(Operation Name)이다.

  • 또한 오퍼레이션 네임 쿼리에서는 변수(Variables)를 사용할 수 있다.

 

3) 인자와 변수 (Arguments & Variables)

  • xxxxxxxxxx
    query HeroNameAndFriends($episode: Episode) {
      hero(episode: $episode) {
        name
        friends {
          name
        }
      }
    }

  • 위 예시에서 $edpisode가 변수이다.

  • ($episode: Episode)는 변수를 선언하는 구문이다. Episode 타입을 갖는 episode라는 변수를 선언한 것이다. 뭐 대충 Dart식으로 보면 Episode episode; 라고 볼 수 있다.

  • hero(episode: $episode)는 hero필드의 episode인자에 $episode 변수를 전달한다는 이야기다.

  • 인자는 서버에서 스키마를 설계하는 단계에서 선언되고, Resolver를 정의하는 단계에서 이 인자가 어떻게 활용될지가 결정된다.

  • 변수는 철저히 클라이언트 사이드(쉽게 말해 쿼리를 보내는 측)에서 선언되고, 할당된다.

  • 변수는 별도의 전송규약(일반적으로 JSON)을 통해 { 변수명 : 할당할 값 }을 담아 보내준다.
