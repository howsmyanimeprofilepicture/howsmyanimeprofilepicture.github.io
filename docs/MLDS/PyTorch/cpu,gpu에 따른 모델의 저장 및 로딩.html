모델의 저장 // GPU, CPU에 따른 전처리

목차

3줄로 요약하는 파이토치 모델 저장 및 불러오기모델이 학습한 parameter의 저장 (state_dict)state_dict 파일 불러오기모델 자체를 저장하기모델 자체를 불러오기CPU / GPU 에 따른 모델 처리GPU로 학습시킨 모델을 CPU로 불러오기CPU로 학습시킨 모델을 GPU로 불러오기 여러 모델 묶음으로 저장하기참조

 

3줄로 요약하는 파이토치 모델 저장 및 불러오기

xxxxxxxxxx
# 경로 지정
PATH = "state_dict_model.pt"


# PATH = "./fff/aa.pt"처럼 상대적 경로로 해도 됨
# 위 예시처럼 파일명만 입력하면 아마 현재 working directory에 있는 모델을 가져오거나 현재 워킹 디렉토리 기준으로 저장될듯
# 위 예시처럼 모델의 확장자는 .pt임
# .state_dict()메서드로 반환하는 state_dict객체가 아닌
# 모델 자체를 save할 수도 있음.


#모델 전체를 저장해도 확장자는 pt임




device = torch.device('cuda:0' if torch.cuda.is_available() else 'cpu') #gpu에선 gpu로 cpu에선 cpu로 device 객체 설정되게 함


# 저장하기
torch.save(net.state_dict(), PATH)


# 불러오기
model = Net()
model.load_state_dict(torch.load(
    PATH,
    device,
))
model.eval()

  • 참고로 net은 모델 인스턴스임...

  • Net 클래스는 커스텀 모델 클래스임

    • 즉 model도 결국 모델 인스턴스란 소리임.

 

모델이 학습한 parameter의 저장 (state_dict)

xxxxxxxxxx
model = torchvision.models.resnet34() #커스텀 모델 클래스
torch.save(model.state_dict(), './afsdf/my_model.pt')

  • 위 예시처럼 model 인스턴스의 .state_dict() 메서드는 해당 모델이 학습한 parameter값이 담긴 state_dict객체를 반환한다.

  • 저장하고자 하는 state_dict객체를 torch.save의 첫번째인자로 전달하고,

  • 두번째 인자로 경로를 설정해서 저장할 수 있다. (확장자는 .pt를 쓴다.)

  • 저렇게 저장하면 해당 파일에 모델이 학습한 가중치와 바이어스들이 저장된다.

    • 모델 자체가 저장되는 것이 아님을 유의하자.

     

     

state_dict 파일 불러오기

xxxxxxxxxx
new_model = torchvision.models.resnet34() #커스텀 모델 클래스
new_model.load_state_dict(torch.load(
    './afsdf/my_model.pt',
    map_location= torch.device('cpu') #여기는 CPU학습모델을 GPU로 불러오나 GPU 학습모델을 CPU로 불러오느냐에 따라 달라짐.
))

  • 위 예시처럼 torch.load로 state_dict객체를 불러오고,

  • 그렇게 불러온 state_dict객체를 model 인스턴스의 .load_state_dict()메서드에 인자로 전달함으로써

  • state_dict에 저장되어 있는 학습 정보(가중치, 바이어스)를 해당 모델에 적용시킬 수 있다.

  • 앞서 말했지만 모델 자체를 불러오는 게 아니다.

  •  

    xxxxxxxxxx
    second_model = torch.nn.Linear(2,1)
    second_model.load_state_dict(torch.load('./afsdf/my_model.pt'))

  • 따라서 위와 같은 코드는 에러가 발생한다.

    • my_model.pt에 저장된 정보는 resnet34 모델의 파라미터들에 대한 학습 정보이다.
    • 하지만 second_model은 단순한 Dense layer이다. 전혀 다른 모델이고, 파라미터 수가 완전히 다르다.
    • 당연히 에러가 날 수밖에 없는 것이다.

  • 따라서 state_dict를 불러오려면

  • .load_state.dict()를 호출하는 모델 인스턴스의 신경망 구조가

  • 불러오고자 하는 state_dict 객체에 담긴 신경망 구조와 일치해야 한다는 소리다.

 

 

모델 자체를 저장하기

xxxxxxxxxx
model = nn.Linear(2,1)
torch.save(model, './afsdf/my_model.pt')
  • 모델 전체를 저장할 수도 있다.

모델 자체를 불러오기

xxxxxxxxxx
model = torch.load('./afsdf/my_model.pt')
  • 해당 모델 인스턴스 전체를 가져올 수 있다.
  • 당연히 가중치와 바이어스에 대한 학습정보도 가져와진다.
  • 모델 신경망 구조를 맞추기가 귀찮으면 이것도 방법일 것 같다.

 

 

CPU / GPU 에 따른 모델 처리

  • 재밌는 사실인데 GPU에서 학습시킨 모델을 불러와서 CPU로 돌리고자한다면 특별한 전처리를 해줘야한다.

  • 반대의 경우도 마찬가지다.

  •  

    xxxxxxxxxx
    device = torch.device('cuda:0' if torch.cuda.is_available() else 'cpu')

    위와 같은 코드를 본적이 있을 것이다.

  • GPU환경에선 device객체가 GPU로 되도록하고, 그게 아니라면 CPU로 설정되게 하라는 코드다.

 

 

GPU로 학습시킨 모델을 CPU로 불러오기

xxxxxxxxxx
# 저장하고자 하는 경로를 지정합니다.
PATH = "./trained_by_gpu.pt"


# 저장하기
torch.save(model.state_dict(), PATH)


# 불러오기
device = torch.device('cpu')
new_model = MyModel()
new_model.load_state_dict(torch.load(PATH, map_location=device))

  • 이경우, Tensor의 저장된 내용은 map_location 인자를 통해 CPU장치에 동적으로 재배치된다.

    • 모델 안에 있는 Tensor말고, 그 밖의 Tensor들도 전부 CPU장치에 맞게 고쳐진다는 소린가?
    • 보면 따로 텐서나 모델에 .to(device)를 할 필요가 없는 것 같다?

 

 

CPU로 학습시킨 모델을 GPU로 불러오기 

xxxxxxxxxx
# 저장하기
torch.save(net.state_dict(), PATH)


# 불러오기
device = torch.device("cuda")
model = Net()
# 사용하고자 하는 GPU 장치 번호를 지정합니다.
model.load_state_dict(torch.load(PATH, map_location="cuda:0"))
# 모델에 사용되는 모든 입력 Tensor들에 대해 input = input.to(device) 을 호출해야 합니다.
model.to(device)

 

  • device 객체를 선언할 때 torch.device("cude:0")처럼 처음부터 GPU 장치 번호를 지정해도 무방한 것 같다.

    • 그렇게 처음부터 지정하면 map_location에 "cuda:0"처럼 str값을 넣지 않고 device객체 torch.device("cuda:0")를 바로 전달해도 무방한 것 같다.

    •  

      xxxxxxxxxx
      model.load_state_dict(torch.load(
          PATH, 
          map_location=torch.device("cuda:0") #이렇게 cude뒤에 GPU번호를 붙이면 하면 device객체를 바로 전달해도 되는듯?
      ))

  • 모델과 그 밖의모든 텐서들에 대해 반드시 .to(torch.device("cuda:0"))을 실행해서 GPU장치에 맞게 변환해야 한다.

    • PyTorch에는 그냥 Tensor와 CUDA Tensor가 있는 것 같다.

 

 

여러 모델 묶음으로 저장하기

xxxxxxxxxx
# 저장할 경로 지정
PATH = "model.pt"


torch.save({
            'modelA_state_dict': netA.state_dict(),
            'modelB_state_dict': netB.state_dict(),
            'optimizerA_state_dict': optimizerA.state_dict(),
            'optimizerB_state_dict': optimizerB.state_dict(),
            }, PATH)
  • 불러올 땐
xxxxxxxxxx
modelA = Net()
modelB = Net()
optimModelA = optim.SGD(modelA.parameters(), lr=0.001, momentum=0.9)
optimModelB = optim.SGD(modelB.parameters(), lr=0.001, momentum=0.9)


checkpoint = torch.load(PATH)
modelA.load_state_dict(checkpoint['modelA_state_dict'])
modelB.load_state_dict(checkpoint['modelB_state_dict'])
optimizerA.load_state_dict(checkpoint['optimizerA_state_dict'])
optimizerB.load_state_dict(checkpoint['optimizerB_state_dict'])


modelA.eval()
modelB.eval()
# - 또는 -
modelA.train()
modelB.train()

 

  • 위 예시처럼 객체 형태로 묶어서 저장하고 불러오는 것도 가능
