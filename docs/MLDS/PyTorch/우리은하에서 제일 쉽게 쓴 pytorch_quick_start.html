1. torchvision.transforms.Compose()

transforms_train = transforms.Compose([
    transforms.Resize((224, 224)),
    transforms.RandomHorizontalFlip(), # 데이터 증진(augmentation)
    transforms.ToTensor(),
    transforms.Normalize([0.485, 0.456, 0.406], [0.229, 0.224, 0.225]) # 정규화(normalization)
])

  • 이렇게 생성된 함수 transforms_train()에

  • PIL.Image.open()으로 불러온 이미지(즉, 바닐라 데이터)를 전달하면,

  • 위 조건에 맞는 텐서로 변환되어 반환된다.

  • torchvision.transforms에 담긴 클래스들은 공통적으로 바닐라 이미지를 텐서로 변환하기 위한 툴들이다.

     

 

2. torchvision.datasets.ImageFolder('./dataset/train_dataset' , transforms_train)

xxxxxxxxxx
./dataset ─ train_dataset ┬ dog ┬ dog1.png
                           │     └ dog2.png
                           ├ cat ┬ cat1.png
                           │     └ cat2.png
                           └ rabbit ─ rabbit1.png

  • datasets.ImageFolder()는 해당 인자로 전달 받은 dir에 있는 이미지들을

  • 폴더명을 기준으로 클래스를 나눈다.

  • 그리고 모든 이미지들에 transform 함수를 적용시켜 전처리도 같이 해준다.

  • 예를 들어 ./dataset/train_dataset 이라는 폴더에 cat, dog, rabbit이라는 세가지 폴더가 있고,

  • cat 폴더에는 cat1.png, cat2.png,

  • dog 폴더에는 dog1.png, dog2.png

  • rabbit 폴더에는 rabbit1.png가 있다고 했을 때

  • 각각의 이미지들이 cat, dog, rabbit에 라벨링된 데이터셋을 만들어준다는 의미다.

  • 라벨링은 당연히 스칼라값(각각의 클래스를 나타내는 인덱스값)으로 되며, 

  • xxxxxxxxxx
    train_datasets = datasets.ImageFolder(os.path.join(data_dir, 'train'), transforms_train)

  • 인덱스를 넣었을 때 반환되는 값은 (sample, target) 형태의 튜플이다. 즉 data와 해당 데이터에 상응하는 label을 튜플로 묶은 값을 반환한다.

    • 이 형태는 결국 DataLoader로 배치를 만들었을 때도 동일해지는데, 결국 배치랑 것은 dataset을 batch_size만큼 행으로 연결한 것이기 때문이다.

 

각각의 인덱스가 어느 클래스에 해당되는지가 궁금하면

  • datasets.ImageFolder()가 반환한 인스턴스의 .class_to_idx 속성으로 확인할 수 있다.
  • 예를 들어 dog 클래스가 0, cat 클래스가 1, rabbit 클래스가 2로 라벨링 되었다면
  • .class_to_idx["dog"] 는 0이고, .class_to_dix["cat"]은 1이 될것이다.

 

 

 

 

3. torch.utils.data.DataLoader

xxxxxxxxxx
train_dataloader = torch.utils.data.DataLoader(train_datasets, batch_size=4, shuffle=True, num_workers=4)

  • DataLoader는 *배치를 표현한 클래스*라고 이해할 수 있다.

    • 따라서 생성자에 전달된 batch_size에 맞춰 iterable한 형태를 띄게 된다.

  • 여기에 전달되는 데이터셋은 transform에 의한 전처리가 끝난 데이터셋이며,

  • batch_size, shuffle 여부, num_workers 정도만 설정해주면 된다.

     

 

4. PyTorch 학습 과정

​x
criterion = nn.CrossEntropyLoss()
optimizer = optim.SGD(model.parameters(), lr=0.001, momentum=0.9)


for inputs, labels in train_dataloader:
        inputs = inputs.to(device)
        labels = labels.to(device)


        # 모델에 입력(forward)하고 결과 계산
        optimizer.zero_grad()
        outputs = model(inputs)
        _, preds = torch.max(outputs, 1)
        loss = criterion(outputs, labels)


        # 역전파를 통해 기울기(gradient) 계산 및 학습 진행
        loss.backward()
        optimizer.step()

 

  • xxxxxxxxxx
    criterion = nn.CrossEntropyLoss()
    optimizer = optim.SGD(model.parameters(), lr=0.001, momentum=0.9)

    • 크라이테리언(손실 함수)은 크로스엔트로피로 했다.

    • 옵티마이저는 확률적경사하강법을 사용한다.

       

     

  • xxxxxxxxxx
    for inputs, labels in train_dataloader:

     

    • 앞서 설명했듯 torchvision.datasets.ImageFolder가 반환하는 데이터셋은 각각의 인덱스 별로 (sample, label) 형태의 값을(튜플) 갖는다.
    • 따라서 for문을 돌릴 때도 for sample, label in dataset:형태가 된다.
    • 결국 배치로 묶었어도 형태는 똑같다.

 

  • xxxxxxxxxx
    optimizer.zero_grad()

    • 정확하게는 모르는데 옵티마이저를 초기화한다는 뜻 같다.

    • 옵티마이저가 뭘까? SGD, ADAM 등등

    • 결국 옵티마이저는 Loss 값을 backpropagation 시켜서 얻은, model.parameters() (weight, bias 등)에 대한 그라디언트 값을

    • 각각의 weight, bias등의 파라미터들에게 차감시킨다는 알고리즘 아닌가?

      • (사실 대부분의 옵티마이저들은 결국은 경사하강법의 변종들이며,
      • 어떻게 LossFunction의 Local Minimum을 피하고,
      • 어떻게 LossFunction의 파라미터들에 대한 기울기가 0이 되는 지점을 찾아갈지에 대한 알고리즘이다.
      • 모든 옵티마이저들은 결국 손실함수와 학습가능한 파라미터들(웨이트앤 바이어스)간의 gradient를 기반으로 동작한다는 뜻이다.

    • 일종의 그라디언트 디센트를 위한 준비가 아닐까 싶다.

       

    • 이전 학습과정에서 구했던 그라디언트가 옵티마이저에 남아있고,

    • 그걸 지워내기 위한 메서드가 아닐까 뇌피셜로 생각해본다.

    • 그래야지 새롭게 구한 그라디언트(loss.backward()를 통해)를 기반으로 다시 그라디언트 디센트는 ADAM이든 최적화를 진행하지 않을까 싶다.

 

  • xxxxxxxxxx
    loss = criterion(outputs, labels)
    • 재밌지만 당연한 사실인데, 모델이 내놓은 outputs는 행렬이고, labels는 벡터이다.
    • 결국 labels에 진짜로 원핫벡터를 할당하는 것은 굉장한 컴퓨팅 낭비이다.
    • labels는 원핫벡터의 인덱스만을 담게 된다. 즉 label자체는 스칼라 값이고, output 자체는 벡터이다.
    • 근데 그걸 배치로 이어 붙였으니, outputs은 행렬이고 되고, labels는 벡터가 되는 것이다.
    • 배치사이즈가 n이고, 카테고리 종류가 m가지일 경우,
    • outputs는 n행m열의 행렬이 되고, labels는 n차원 벡터가 될 것이다.

 

 

  • xxxxxxxxxx
    loss.backward()
    optimizer.step()

  • loss.backward()는 크라이테리언(손실함수)가 구한 loss를 백프로파게이션 시킨다.

    • 이를 통해 Loss Function의 각각의 파라미터에 대한 편미분값(∂Loss(θ)∂θ)들을 계산한다. (즉, 그라디언트가 계산된다는 뜻이다.)
    • 이 과정에서 각 파라미터(weight, bias 등...) 텐서에 저장되어 있던 그라디언트값이 연쇄법칙(chain-rule)에 활용된다.

     

  • 그리고 optimizer.step()을 통해 각각의 파라미터(θ)에 앞서 구한 편미분값(∂Loss(θ)∂θ)들을 빼준다.

    • 그냥 뺴는 것은 아니고, 옵티마이저에 따라 모멘텀이나 다양한 추가 조정이 들어간다.
    • 근데 결국은 각각의 파라미터에 편미분값(∂Loss(θ)∂θ)을 차감하는 것은 동일하다.
    • 앞서 말했듯 Adam이든 SGD든 모든 옵티마이저는 결국 경사하강법의 변종들이기 때문이다.
    • 다만 로컬미니멈에 빠지지 않고, 좀 더 효율적으로 손실함수의 미니멈 값을 찾기 위해 조금씩 다른 알고리즘이 동원되었을 뿐이다.

 

 

  • 왜 손실함수의 그라디언트가 0이 되는 지점이 손실함수가 최소가 되는 값, 즉 최적의 값인지는 아래 이미지를 참고하자.
  • 위 함수의 최소값의 기울기는 몇인가? 0이다. 설명 끝!
