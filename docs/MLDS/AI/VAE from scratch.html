뇌피셜

  • VAE모델에서 Encoder가 내놓는 output은 input data의 확률분포의 모수(parameter)이다. input이 이항분포라면 p(시행결과가 성공할 확률)가 될 것이고, input이 정규분포라면 평균과 분산을 인코더가 내놓을 것이다.

  • 인풋이 단색비트맵 이미지라면 즉 각 픽셀이 완전한 검은색과, 완전한 흰색만 존재한다면 인풋은 이항분포를 뛴다고 해석할 수 있을 것이다.

  • 이 때 해당 이미지의 어떤 한 픽셀이 검은색일 확률을 p라고 한다면,

  • input data의 실제 확률 x와 decoder가 산출한 확률 p 사이의 크로스엔트로피를 에러 펑션으로 쓸 수 있을 것이다.

    • 크로스 엔트로피 자체가 실제확률과 추정확률 사이의 차이로 발생하는 엔트로피를 의미하므로

  • input이 정규분포를 따른다면 decoder는 평균과 분산을 내놓을 것이다.

    • 이 때 input의 실제 평균, 분산과 디코더가 내놓은 추정 평균, 분산 사이의 오차값을 에러펑션으로 사용할 것이다. 즉 MSE(평균 제곱 오차)가 사용될 것이란 얘기다.

 

첫번째 Error Function : Reconstruction Error

  • H(xi,pi)=−∑j=1Dxi,jlog⁡pi,j+(1−xi,j)log(1−pi,j)

    • xi,j는 결국 인풋이미지의 각픽셀에 잉크가 묻어있을 실제확률을 의미한다.
    • pi,j는 latent vector를 디코더에 넣어 생성한 이미지의 각픽셀에 잉크가 묻어 있을 확률을 의미한다.
    • 각 픽셀은 베르누이 분포를 따르는데,
    • 각 픽셀은 잉크가 묻었거나(1) 잉크가 안 묻었거나(0)로만 구분되기 때문이다.
    • 잉크가 얼마나 진하게 묻었는가에 대해서는 생각하지 않기에 베르누이 분포를 따르게 된다.
    • 만약 잉크가 0~9까지 명도를 구분하게 된다면
    • 그 안에서 평균값과 분산값이 존재하는 어떠한 분포를 보일 것이다.

 

 

 

 

크로스 엔트로피가 기억이 안난다면

H(P,Q)=−∑i=1kPilog⁡Qi

  • P는 실제확률

  • Q는 예측활률

  • P1+P2+⋯+Pk=1

  • Q1+Q2+⋯+Qk=1

  • −log⁡Qi는 예측 확률이 갖는 정보의 양을 의미한다 이해할 수 있고,

    • 이에 대해서 첨언하자면 로그함수 그래프를 보면 좋다.
    • 파란색 선을 보면 쉬운데,
    • 밑이 몇이 되건 로그함수는 x가 1일 때 0이 된다.
    • 즉 100% 일어나는 일의 정보의 양은 0이라는 이야기가 된다.
    • 확률이 낮아질 수록 0에 근접할 수록 로그함수는 무한에 근접하게 되는데,
    • 낮은 확률로 벌어지는 일일수록 높은 정보의 양을 지님을 의미한다.
    • 그냥 엔트로피(H)는 이러한 각 사건의 정보의 양을 해당 사건이 지닌 확률에 따라 가중평균을 구해준 것이라 할 수 있다. 달리 생각하면 정보의 양의 기대값 정도로 이해해도 무방하다.

  • 그걸 Pi로 각각 곱해주는 것은 결국 예측확률(Q)이 갖는 정보의 양의 기대값, 혹은 가중평균을 구하는 것으로 이해할 수 있다.

  • 공대생 수학정리 블로그에선 실제확률과 예측확률 달라 생기는 깜놀도라고 표현한다.

    정규분포의 확률밀도함수

  • 유도하다 귀차나서 포기함

  • 정규분포와 큐파이 지 기븐 엑스 아이의 KL 다이버전스 두 분포의 차이를 0으로 만들어야 함

 

 

 

VAE의 핵심 아이디어

 

  • pθ(x)=∫−∞∞pθ(z)pθ(x|z)dz

    • 위 수식은 통계학적으로 수학적으로 당연히 성립하는 수식이다.

    • 모든 z에 대해 x|z의 확률을 합한 것이기 때문이다.

    • pθ(z)는 우리가 그저 가우시안(정규분포)을 따른다고 가정한다.

    • pθ(x|z)는 디코더 신경망의 확률 분포를 표현한 것으로 이해할 수 있다.

    • qΦ(z|x) encoder는 pθ(z|x)를 근사화한 네트워크

      • Encoder 자체가 z와 decoder만으로는 모델링이 불가능해서 생각해낸 개념으로 받아드리면 쉬움
      • p(z)와 p(x|z)만으로는 p(x)를 생성(generate)해낼수가 없었기 때문임.

    • 결국 목적은 디코더고, 디코더를 구현하기 위해 인코더를 만들어낸 것
