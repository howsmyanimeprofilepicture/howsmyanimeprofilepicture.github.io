변수(variables)와 모수(parameters)

  • 프로그래밍에서의 함수 패러미터처럼 생각하면 큰 오산이다. 비슷하면서도 매우 다르다. 정말 매우 다르다.

  • Likelihood Function(가능도 함수) L(θ;x)의 함수값은 x(모집단에서 추출한 표본들)가 아닌 θ(모집단의 모수)에 의해 값이 결정된다. f(x)가 아닌 f(θ)꼴의 함수라는 이야기다.

    • x는 변수가 아니다. 모집단에서의 표본 추출은 이미 결정된 사항들이며, x는 이미 결정된 값들이다. 즉, 모수(parameters)는 이미 결정된 값들을 의미한다.
    • 가능도 함수 L(θ;x)은 θ(모집단의 모수)값의 변화에 따른 가능도를 측정하는 것이 목적이다. 따라서 가능도 함수 L(θ;x)의 변수는 x가 아닌 θ가 되는 것이다.

  • 확률밀도함수(혹은 확률질량함수)는 반대로 θ가 모수가 된다. 확률밀도함수는 결국 확률변수x에 대한 확률을 함수값으로 갖기 때문이다.

  • 패러미터가 사용되는 함수는 function name (variables ; parameters) 처럼 표시한다

  • 정규분포 확률밀도함수를 예로 들면 여기서 변수는 x가 되고 모수는 μ와 σ가 된다.

     

 

 

가능도란 무엇인가?

  • 가능도, 우도라고 번역되는 거 같다.
  • 사전에서 Likelihood는 가능성 정도로 번역한다.

  • 통계학에서 가능도(Likelihood)는 함수로 표현할 수 있다.

    • 가능도 함수는 L(θ ; x)로 보통 표현된다.

    • x는 모집단으로부터 추출한 표본들의 집합이다.

    • θ는 모집단의 모수들의 집합이다.

    • 가능도 함수는 모집단의 모수 θ를 변수로 한다.

      • 가능도 함수의 모수(paremeter)는 모집단으로부터 추철한 표본들의 집합 x가 된다.

    • 즉 모집단의 모수 θ가 어떤 값이냐에 따라, 해당 모집단에서 표본 x를 발견할 가능성을 수치화한 개념이 바로 가능도라 할 수 있다.

  • θ를 모수로하는 어떠한 확률분포를 따르는 모집단이 있고, 거기서 5개의 표본을 추출했다고 가정하자.
  • 그 표본은 x = {1, 4, 5, 6, 9} 가 된다.
  • 주황색 곡선은 θ1을 모수로 하는 확률분포 f(x ; θ1)이고,
  • 파란색 곡선은 θ2를 모수로 하는 확률분포 f(x ; θ2)이다.
  • 표본 x는 주황색 곡선과 파란색 곡선 중 어디서 추출되었을 가능성이 더 높을까?
  • 직관적으로 주황색 곡선에서 표본 x를 추출할 가능성이 더 높다는 것을 알 수 있다.
  • 이 때 모집단의 모수가 θ1일 때 가능도가 더 높다고 표현할 수 있다.
  • 이를 함수로 표현하면 L(θ1; x)은 f(1; θ1)+f(4; θ1)+f(5; θ1)+f(6; θ1)+f(9; θ1)가 될 것이고,
  • L(θ2; x)는 f(1; θ2)+f(4; θ2)+f(5; θ2)+f(6; θ2)+f(9; θ2)이 될 것이다.

 

 

가능도(likelihood)는 확률(probablity)이 아니다.

  • 확률밀도함수(혹은 확률질량함수)를 -∞에서 ∞까지 적분하면, 1이 나온다.
  • 가능도 함수(L(θ;x))를 -∞에서 ∞까지 적분하면? 절대 1이 나오지 않을 것이다.
  • 가능도 ≠ 확률인 이유이다.

 

 

MLE, Maximum Likelihood Estimation

  • MLE는 모수적인(parametic) 추정법이다.

  • 즉 모집단이 어떠한 분포를 따를지에대해 선험적인(a priori) 가정이 필요하다.

  • 모집단이 정규분포를 따른다고 가정한다면 우리가 찾는 θ는 모집단의 평균(μ)과 분산(σ2)이 될 것이다.

  • θMLE = argmaxθ L(θ;x) = argmaxθ P(x|θ)

    • 가능도함수 L(θ;x)의 함수값을 최대치로 만드는 θ값을 의미
    • 조건부확률 P(x|θ)을 최대치로 하는 θ값
    • 이 둘은 결국 같은 개념으로 이해해도 좋을 것이다.

 

 

 

로그가능도

  • 모집단은 f(x; θ)의 분포를 따르며, 해당 모집단에서 추출한 표본들의 집합은 x = {x1, x2, ... , xn} 와 같다.

  • 그냥 가능도 함수는 f(x1; θ) + f(x2; θ) + ... + f(xn; θ)로 표현할 수 있을 것이다.

  • 로그가능도는 loge[f(x1; θ)] + loge[f(x2; θ)] + ... + loge[f(xn; θ)]로 표현할 수 있다.

  • 뇌피셜인데

    • y=x나 y=logex나 우상향하는 그래프인 건 동일하다.
    • 정확한 가능도값이 알고 싶은 것이 아니라 단순히 가능도의 최대값을 찾는 것이라면 로그가능도를 사용하여도 무방할 것이다.
    • 정규분포의 확률밀도함수 같은 경우, 로그가능도 함수를 사용하면 계산이 훨씬 편해진다.

 

  • 위 내용은 아직까지 완벽히 이해하지 못했다.

 

 

정규분포를 대상으로 MLE 사용하기

  • 모집단이 정규분포를 따른다면, 모집단의 모수(θ)는 평균(μ)과 분산(σ2)이 될 것이다.

  • 모집단의 확률밀도함수f(x ; μ, σ)는 위와 같이 될 것이다.

  • 계산의 편의를 위해 log를 씌어주자... (참고로 log의 밑이 생략되었다면 loge로 생각하면 된다.)

  • logef(x ; μ, σ) = -(1/2)*loge(2π) - logeσ - (x-μ)2/2σ2

  • 모집단에서 추출한 표본 집합은 x={x1, x2, ... , xn}과 같다.

  • 가능도함수는 L(μ, σ ;x) = logef(x1 ; μ, σ) + logef(x2 ; μ, σ) + ... + logef(xn ; μ, σ) 과 같이 표현할 수 있다.'

    • 가능도함수는 μ과 σ라는 2개의 변수를 가지므로, 각각의 변수에 대한 편미분이 필요하다.
    • μ와 σ로 각각 편미분한 도함수가 0이 되는 μ와 σ를 찾으면 된다.
    • 도함수가 0이 되었다는 것은 이때의 변수값이 argmax라는 뜻이다.

  • 수학적으로 모두 쓰려고 하니 너무 더러워져서... 급 의욕이 떨어졌다.

  • 공책에 스스로 풀어보면 아래와 같이 나온다는 것을 알 수 있을 것이다.
