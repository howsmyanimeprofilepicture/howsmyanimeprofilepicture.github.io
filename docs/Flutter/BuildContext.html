1. BuildContext 객체의 이해

class MyApp extends StatelessWidget {
    
  @override
  Widget build(BuildContext context) {
      //build함수의 매개변수 context는  
      //함수를 호출하는 위젯의 BuildContext이다.
      
      //build함수가 리턴하는 위젯의 
      //BuildContext가 아니다! 
      return Container(
      child: MaterialApp(
        home: TasksScreen(),
      ),
    );
  }
    
  1. 모든 위젯은 BuildContext를 지닌다.
  2. 단 BuildContext에 접근할 수 있느냐는 다른 문제이다.
  3. StatelessWidget, Builder 위젯, State 객체 등은 build 메서드를 갖고 있기 때문에 해당 위젯의 context가 build메서드의 매개변수( BuildContext context)를 통해 노출된다. 하지만 다른 위젯들은 그렇지 않다.
  4. StatelessWidget이나 State객체의 BuildContext는 해당 위젯의 build 메서드의 매개변수가 된다..
  5. Builder 위젯은 위젯트리 중간에 '참조할 수 있는' context를 만들고자 할 때 사용한다.
  6. 예를 들어 스낵바 메시지를 사용하기 위해 Scaffold위젯을 참조해야 하는데 Scaffold위젯을 포함하는 context가 없을 때 Builder 위젯을 통해 context를 만들어보는 것은 아주 좋은 해결책이다.

2. of(BuildContext context) 메서드와 조합하기

  1. of(context) 특정 위젯의 위치를 참조하기 위한 메서드다.
  2. 인자로 전달된 context를 기준으로, 위젯트리에서 상위에 위치한 위젯중에 가장 가까운 위젯을 리턴한다.

 

2.1. Scaffold.of(context)

  1. 위 예제는 of메서드에 전달된 context를 기준으로, 위젯트리상 상위에 위치한 위젯 중에 가장 가까운 Scaffold 위젯을 리턴한다.
  2. .showSnackBar(SnackBar(content: Text('ㅎㄹㅇㅎㅇㄹ'),),)를 통해 스낵바 메시징을 구현할 수 있다.

 

2.2. Theme.of(ctx)

Theme 위젯과 ThemeData 위젯에 대한 공부가 더 필요할 것 같다.

 

Dart는 모든 것이 객체다. 모든 자료타입은 클래스이고, 모든 값은 어떠한 클래스의 인스턴스이다. 뭐 1이라는 정수를 예로 들면 1도 결국 Int라는 Class의 인스턴스일 뿐이라는 얘기다. Widget 역시 클래스이고, Scaffold 역시 Widget이라는 클래스를 상속받은 클래스이다.

 

 

2.3. MediaQuery.of(ctx)

  1. 인자로 전달된 ctx를 기준으로 가장 가까운 MediaQueryData를 찾는다.
  2. 인자로 전달되는 빌드컨텍스트는 위젯트리상 MeterialApp위젯(혹은 CupertinoApp, WidgetsApp)보다 하위에 있어야 한다.
  3. 왜냐하면 MediaQuery객체는 MaterialApp위젯을 통해 전달되기 때문이다. (이거 불확실함..)
