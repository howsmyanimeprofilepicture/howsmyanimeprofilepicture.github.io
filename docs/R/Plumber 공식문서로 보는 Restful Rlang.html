#* Echo the parameter that was sent in
#* @param msg The message to echo back.
#* @get /echo
function(msg=""){
  list(msg = paste0("The message is: '", msg, "'"))
}
​
#* Plot out data from the iris dataset
#* @param spec If provided, filter the data to only this species (e.g. 'setosa')
#* @get /plot
#* @serializer png
function(spec){
  myData <- iris
  title <- "All Species"
​
  # Filter if the species was specified
  if (!missing(spec)){
    title <- paste0("Only the '", spec, "' Species")
    myData <- subset(iris, Species == spec)
  }
​
  plot(myData$Sepal.Length, myData$Petal.Length,
       main=title, xlab="Sepal Length", ylab="Petal Length")
}
​
  • 조금 충격적인 건 #*라는 프리픽스(prefix)의 사용이다. 이게 문법적으로 어떻게 가능한 것인지 싶다. 주석에도 어떠한 프로그래밍적 로직을 담을 수 있다는 얘긴가?
  • 아마도 #*는 R 언어에서의 데코레이터 표기인 것 같다.
  • @get은 HTTP메서드를 의미한다.
  • @param은 함수의 파라미터를 쿼리스트링으로 사용함을 의미한다
  • @serializer는 엔드포인트 함수가 리턴하는 값, 즉 응답(res)의 타입을 결정한다. 따로 @serializer가 명시되어 있지 않다면 JSON으로 응답한다. @serializer html @serializer jpeg @serializer png
