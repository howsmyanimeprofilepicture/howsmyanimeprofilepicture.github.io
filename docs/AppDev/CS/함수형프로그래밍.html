1. 함수형 프로그래밍(Functional Programming)

 

  • 대입문이 없다.
  • 순수 함수
  • SideEffect가 없다.
  • 함수와 데이터를 중점으로 사고한다.

 

순수함과 부작용

함수형 프로그래밍의 관점에서 생각해보자.

FP(Functional Programming) 근본주의적 관점에서

함수란 입력을 받아 그것을 기반으로 값을 출력(반환)하는 것이다.

전달받은 파라미터가 아닌 외부 변수를 조작한다면,

그것은 순수하지 못하며, 부작용(SideEffect)을 갖는 함수란 이야기다.

 

 

 

2. 왜 함수형 프로그래밍인가?

  • 기본적으로 예상 가능한 프로그래밍을 위해 고안된 프로그래밍 방법론이다.

  • 예측 불가능한 오류를 최소화할 수 있다.
