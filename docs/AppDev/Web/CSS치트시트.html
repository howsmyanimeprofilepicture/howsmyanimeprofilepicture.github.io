1. position

1.1. static

  • 모든 태그들은 처음에는 position: static 상태이다.
  • 요소를 일반적인 문서 흐름에 따라 배치합니다
  • top, right, bottom, left, z-index가 적용되지 않는다.

1.2. relative

  • top, right, bottom, left를 적용하여 위치를 조절할 수 있다.
  • static일 경우의 위치를 기준으로, 주어진 top, right, bottom, left 값만큼 이동한다.

1.3. absolute

  • position이 static이 아닌 부모를 기준으로 위치를 조정할 수 있다. (이게 relative와 차이임.)
  • 부모의 포지션이 static이면 부모의 부모의 position을 체크한다.

1.4. fixed

  • 뷰포트를 기준으로 배치한다.
  • 더 정확히는 뷰포트의 초기 컨테이닝 블록을 기준으로 배치한다.

1.5. sticky

  • 평소에는 static하게 위치가 정해진다.
  • top, right, bottom, left를 적용하여 위치를 fixed 시킬 수 있다.
  • 써보는 것이 가장 직관적일 것이다.

3. 함수와 변수

  • attr()

    • 인자로 전달된 요소(attribute)의 값을 반환한다.

    • a:after {
          content: "(" attr(href) ")";
      }

    • 이렇게 되면 a태그 뒤에 링크주소(href의 값)가 소괄호에 감싸져서 나온다.

  • calc()

    • 사칙 연산 수행, 서로 다른 단위로도 가능함.

    • div {
          width: calc(100% - 80px);
      }

  • var()

    • xxxxxxxxxx
      :root {
          --main-bg-color: pink;
      }
      ​
      body {
          background-color: var(--main-bg-color);
      }

5. display

  • none

    • 렌더링되지 않게 함. 안 보임

  • block

    • 엘리먼트들이 세로로 배치된다.

    • width, height 값이 모두 적용된다.

    •  

  • inline

    • width, height가 적용되지 않는다.
    • 가로로 배치된다.

  • inline-block

    • 가로로 배치되는 면은 inline을, width, height가 적용되는 모습은 block을 닮았다 할 수 있다.
