1. Side Effect

​x
function makeSideEffect(obj) {
    obj.number ++;
    return obj;
};
​
const a = {number : 1};
const b = makeSideEffect(a);
console.log(`a : ${JSON.stringify(a)}`)
console.log(`b : ${JSON.stringify(b)}`)
a.number ++;
console.log(`a : ${JSON.stringify(a)}`)
console.log(`b : ${JSON.stringify(b)}`)
​

실행결과

xxxxxxxxxx
a : {"number":2}
VM1003:9 b : {"number":2}
VM1003:11 a : {"number":3}
VM1003:12 b : {"number":3}

 

  • 위 예시의 makeSideEffect함수는 순수하지 않으며, Side Effect를 발생시킨다.

  • 비록 인자로 전달받았다고는 하나 외부변수인 객체 a를 수정하기 때문이다.

  • 그리고 인자로 전달받은 객체를 그대로 리턴함에 따라 동일한 객체를 변수 a와 변수 b가 공유하게 된다.

  • 여기서 makeSideEffect의 인자(obj)와 변수 a가 다른 객체라고 이해할 수도 있을 거 같은데...

  • 숫자나 문자열같은 원시값이라면 그게 맞는 이야기가 될 수 있다. 하지만 객체가 인자로 전달될 경우, 변수에는 객체의 메모리 주소가 할당되는 것이기 떄문에, 함수의 인자(obj)나 거기에 할당된 변수 a나 같은 객체를 참조하는 것이다.

    xxxxxxxxxx
    const plus = num => {
        num = num+1;
        return num;
    }   
    let num1 = 1;
    let num2 = plus(num1);
    console.log(num1, num2);
    // 1, 2,

    이와 같이 원시값이 할당된 경우에는 파라미터와 인자로 들어간 변수가 별개로 취급된다.

  • 사이드 이펙트를 발생시키지 않는 함수를 순수 함수라고 한다.

 

 

2. 참조 투명성

xxxxxxxxxx
 int globalValue = 0;
​
 int rq(int x) {
   globalValue++;
   return x + globalValue;
 } // 불투명함
​
 int rt(int x) {
   return x + 1;
 } // 투명함
​

  • 공부가 더 필요한데, 대충 함수를 반복 실행해도 같은 결과를 반환하면 참조가 투명하다고 보는 것 같다.

    1+1은 언제나 1이다. 근데 외부 변수에 기반한 함수는 반복 실행시 같은 결과를 보장할 수 없다.

  • 외부 변수에 의존하지 않고, 매개 변수와 내부 변수만으로 해결한다면, 참조투명성을 지킬 수 있다.
